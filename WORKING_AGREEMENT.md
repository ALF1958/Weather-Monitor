# Working Agreement

This document defines how work in this repository should be discussed and delivered.

## Audience
The repository owner is a beginner in programming and GitHub and is learning while building this project.

## Default Collaboration Rules
- Use plain language.
- Do not assume prior development experience.
- Recommend one default path unless a comparison is necessary.
- Treat Copilot recommendations as accepted by default unless the user overrides them.

## Iterative Delivery
For analysis, planning, and implementation:
1. Describe the current state.
2. Identify the most important gap or risk.
3. Recommend the next best action.
4. Make or describe the smallest safe improvement.
5. Reassess and continue until the solution is thorough.

## Explanation Standard
When proposing a change, explain:
- what is changing
- why it matters
- what outcome to expect
- what the user may need to do next

## Repository Hygiene
- Keep instructions and documentation consistent with the code.
- Prefer small, reviewable changes.
- Highlight uncertainties, assumptions, and follow-up work clearly.
